## 5. Mercado Angolano - Análise Específica

### 5.1 Contexto Financeiro

| Métrica | Valor |
|---------|-------|
| População sem conta bancária | ~60% |
| Penetração de pagamentos móveis (2022) | 65% |
| Utilizadores Unitel Money (2022) | 3+ milhões |
| Crescimento de transações móveis | +50% YoY |

### 5.2 Métodos de Pagamento Locais

| Serviço | Tipo | Descrição |
|---------|------|-----------|
| **Unitel Money** | Mobile Money | Líder de mercado, 3M+ utilizadores |
| **Multicaixa Express** | Mobile Banking | Sistema bancário nacional |
| **e-Kwanza** | Mobile Money | Serviço do BAI |
| **AkiPaga** | Mobile Money | Solução emergente |
| **Zap** | Mobile Money | Alternativa popular |

### 5.3 Oportunidades no Mercado Angolano

1. **Inclusão Financeira:** 60% da população sem conta bancária = oportunidade para mobile money
2. **Mobile-First:** Alta penetração de smartphones vs computadores
3. **Turismo em Crescimento:** Angola como destino emergente
4. **Poucos Concorrentes Locais:** Mercado subdesenvolvido em apps de reserva

### 5.4 Desafios

- Infraestrutura de internet inconsistente
- Necessidade de modo offline
- Múltiplas moedas (AOA, USD)
- Verificação de identidade para pagamentos

## 6. Definição do MVP - Minimum Viable Product

### 6.1 O Que é MVP?

O **Minimum Viable Product (MVP)** é a versão mais simples do produto que permite:
- Validar a proposta de valor com utilizadores reais
- Recolher feedback antes de investir em funcionalidades complexas
- Lançar rapidamente no mercado
- Minimizar riscos e custos de desenvolvimento

### 6.2 Critérios para MVP do StayGo

O MVP deve responder a estas perguntas fundamentais:

| Pergunta | Funcionalidade Necessária |
|----------|---------------------------|
| O utilizador consegue encontrar um hotel? | ✅ Pesquisa e Filtros |
| O utilizador consegue ver detalhes do hotel? | ✅ Página de Detalhe |
| O utilizador consegue fazer uma reserva? | ✅ Fluxo de Booking |
| O utilizador consegue pagar? | ✅ Sistema de Pagamentos |
| O utilizador recebe confirmação? | ✅ Confirmação de Reserva |
| O utilizador consegue ver suas reservas? | ✅ Histórico de Bookings |

### 6.4 MVP - User Stories

#### Utilizador Anónimo (Pré-Registo)

```
Como visitante,
Quero pesquisar hotéis por localização,
Para que possa ver as opções disponíveis.

Como visitante,
Quero ver detalhes de um hotel,
Para que possa decidir se é adequado para mim.

Como visitante,
Quero criar uma conta facilmente,
Para que possa fazer uma reserva.
```

#### Utilizador Registado

```
Como utilizador registado,
Quero fazer login com meu telefone,
Para que possa aceder à minha conta rapidamente.

Como utilizador registado,
Quero selecionar datas de check-in e check-out,
Para que possa reservar para as datas corretas.

Como utilizador registado,
Quero escolher o número de hóspedes,
Para que o hotel saiba quantas pessoas vão ficar.

Como utilizador registado,
Quero ver o preço total antes de pagar,
Para que não haja surpresas.

Como utilizador registado,
Quero pagar com Unitel Money,
Para que possa usar o método mais conveniente para mim.

Como utilizador registado,
Quero receber confirmação da minha reserva,
Para que tenha prova do booking.

Como utilizador registado,
Quero ver todas as minhas reservas,
Para que possa gerir minhas viagens.

Como utilizador registado,
Quero guardar hotéis favoritos,
Para que possa encontrá-los facilmente depois.

Como utilizador registado,
Quero cancelar uma reserva,
Para que possa mudar de planos se necessário.
```

## 13. Conclusões e Recomendações

### 13.1 Oportunidades Principais

1. **Mercado Mobile em Crescimento:** 60% das reservas são mobile - posicionamento correto
2. **Mercado Angolano Subdesenvolvido:** Poucos concorrentes locais especializados
3. **Mobile Money:** Penetração de 65% - oportunidade única de inclusão financeira
4. **Personalização:** 68% querem experiências personalizadas - diferenciador

### 13.2 Ações Prioritárias

1. **Imediato (MVP - 0-4 meses)**
   - Desenvolver backend e API
   - Implementar autenticação de utilizador (Phone + Email)
   - Integrar Unitel Money e pagamento por cartão
   - Configurar serviço de emails
   - Criar ecrãs de onboarding
   - Submeter às app stores

2. **Curto Prazo (4-6 meses)**
   - Lançar push notifications
   - Implementar sistema de reviews
   - Estabilizar e corrigir bugs
   - Configurar analytics

3. **Médio Prazo (6-9 meses)**
   - Lançar programa de fidelidade básico
   - Implementar chat/suporte in-app
   - Adicionar modo offline
   - Expandir para multi-idioma

4. **Longo Prazo (9-12 meses)**
   - AI-powered recommendations
   - Check-in digital
   - Tours virtuais
   - Experiências locais

### 13.3 Diferenciadores Competitivos Sugeridos

| Diferenciador | Descrição | Impacto Esperado |
|---------------|-----------|------------------|
| **Mobile Money First** | Integração nativa com Unitel Money, Multicaixa | Alcance de 60% da população sem banco |
| **Modo Offline** | Funcionalidade básica sem internet | Usabilidade em áreas rurais |
| **Suporte em Português** | App e suporte totalmente em PT-PT/BR | Confiança do utilizador local |
| **Preços em AOA** | Evitar confusão cambial | Transparência |
| **Parcerias Locais** | Hotéis angolanos exclusivos | Inventário único |

### 13.4 Próximos Passos Recomendados

1. **Validar o MVP:**
   - Entrevistar 10-20 potenciais utilizadores
   - Validar proposta de valor
   - Confirmar que funcionalidades core fazem sentido

2. **Construir Backend:**
   - Definir stack tecnológico (Node.js/Python + PostgreSQL recomendado)
   - Desenhar arquitectura de API
   - Implementar autenticação segura

3. **Parcerias com Hotéis:**
   - Identificar 10-20 hotéis para lançamento
   - Negociar comissões
   - Definir processo de onboarding

4. **Integração de Pagamentos:**
   - Contactar Unitel Money para parceria
   - Implementar gateway de cartões (Stripe/PayStack)
   - Testar fluxos de pagamento

5. **Preparar Lançamento:**
   - Criar material de marketing
   - Preparar App Store e Play Store listings
   - Definir estratégia de aquisição de utilizadores
